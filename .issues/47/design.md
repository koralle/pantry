# 画面パス設計（Issue #47）

## 概要

自分専用タグベースブックマークマネージャ（pantry）の画面 URL 設計を定める。
MVP で必要な画面は「一覧 / 詳細 / 登録 / 編集 / 設定」に加え、既存の「タグ一覧 / 認証」画面とタグの登録・編集画面を含む。

## 設計方針

- `/` をブックマーク一覧（ホーム）の正規 URL（canonical）とする
- 未認証時は保護対象パスへのアクセスを `/sign-in` へリダイレクトする
- 新規作成と編集は別パスとし、ブラウザの「戻る / 進む」を自然に保つ
- 一覧の検索・絞り込み・並び順はクエリパラメータに持たせ、URL として共有・ブックマーク可能にする
- `/bookmarks` は `/` へのエイリアスとして 301 リダイレクトする

## 画面パス一覧

### 認証系（既存維持）

| パス          | 画面            | 備考               |
| ------------- | --------------- | ------------------ |
| `/sign-in`    | ログイン        | 未認証用           |
| `/sign-up`    | ユーザー登録    | 未認証用           |
| `/api/auth/$` | Better Auth API | 認証エンドポイント |

### ブックマーク系

| パス                  | 画面                      | 備考                      |
| --------------------- | ------------------------- | ------------------------- |
| `/`                   | ブックマーク一覧          | canonical ホーム          |
| `/bookmarks`          | `/` への 301 リダイレクト | 手打ち / 推測用エイリアス |
| `/bookmarks/new`      | ブックマーク新規作成      |                           |
| `/bookmarks/:id`      | ブックマーク詳細          | `:id` は UUID v7          |
| `/bookmarks/:id/edit` | ブックマーク編集          |                           |

### タグ・設定系

| パス             | 画面           | 備考                           |
| ---------------- | -------------- | ------------------------------ |
| `/tags`          | タグ一覧       | 既存画面を維持                 |
| `/tags/new`      | タグ新規登録   | ページシェル（機能は別タスク） |
| `/tags/:id/edit` | タグ編集       | ページシェル（機能は別タスク） |
| `/settings`      | アカウント設定 | Better Auth 連携状態確認を含む |

## 一覧画面のクエリパラメータ

`/` および `/bookmarks`（リダイレクト後）で受け付ける検索・絞り込み・並び順パラメータ。

| パラメータ | 型                    | 例                               | 用途                                       |
| ---------- | --------------------- | -------------------------------- | ------------------------------------------ |
| `q`        | string                | `?q=react`                       | キーワード検索（title / url / note）       |
| `tags`     | string[]              | `?tags=frontend&tags=typescript` | タグ絞り込み。繰り返し形式のみ             |
| `tagMode`  | `and` \| `or`         | `?tagMode=and`                   | タグ条件の AND / OR 切替。未指定時は `and` |
| `sort`     | `newest` \| `updated` | `?sort=updated`                  | 並び順。未指定時は `newest`                |

### 例

```
/?q=react&tags=frontend&tags=typescript&tagMode=and&sort=updated
```

## リダイレクト挙動

### `/bookmarks` → `/` のリダイレクト

- SSR 時に HTTP 301 を返し `/` へリダイレクトする
- ブラウザ履歴に `/bookmarks` を残さないため、バックトラップを防ぐ
- `@tanstack/react-start` の `redirect` を利用し、`statusCode: 301` を指定する
- リダイレクト用の `/bookmarks` ルートは `_protected` レイアウト配下に配置し、未認証時は認証リダイレクトが先に発生する
- 認証済み時に `/bookmarks` を開いた場合のみ、301 で `/` へ遷移する

### 未認証時のリダイレクト

- 保護対象パス（`/_protected` 配下）へ未認証でアクセス → `/sign-in` へリダイレクト
- リダイレクト時は `callbackURL` として元のパスを保持し、ログイン後に元の画面へ戻す
- ログイン成功後の遷移は `replace: true` で行い、「戻る」で `/sign-in` に戻らないようにする

## ブラウザバックとの整合性

今回のパス設計はブラウザの「戻る / 進む」と整合性が高い。

- **別ページ構成**（`/bookmarks/new`、 `/bookmarks/:id/edit`、 `/tags/new`、 `/tags/:id/edit`）により、保存後の「戻る」が自然に前画面に戻る
- **クエリパラメータでフィルタ状態保持**により、タグ追加や検索変更を「戻る」で段階的に元に戻せる
- **301 リダイレクト**により、 `/bookmarks` 経由で `/` に来た後も「戻る」が `/bookmarks` に捕まらない

## 実装時の検証項目

- `@tanstack/react-start` の `redirect` が Cloudflare Workers 環境で 301 レスポンスを返すことを確認
- `/bookmarks` リダイレクトと `_protected` レイアウトの認証リダイレクトが二重に発生しないことを確認
- 未認証時の deep link（例: `/bookmarks/xxx/edit`、 `/tags/1/edit`）が、ログイン後に元の画面へ戻ることを確認
- 一覧画面のクエリパラメータがブラウザリフレッシュ後も復元されることを確認
